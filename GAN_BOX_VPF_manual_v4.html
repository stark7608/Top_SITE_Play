코드 검토 결과 — 개발자 전달
전달된 코드는 개발 방향과 90% 일치하며 완성도가 높습니다. 단, 핵심 신호 로직에 영향을 주는 수정 사항 2건이 있습니다. 배포 전 반드시 적용해주세요.
🔴 CRITICAL — 수정 필수 (신호 정확도에 직접 영향)
[1] VPF 기울기 봉수: 1봉 → 5봉
vpf_slope = vpf - nz(vpf[1]) // 현재: 1봉 차이 (노이즈 심함)
vpf_slope = vpf - nz(vpf[5]) // 수정: 5봉 차이 (백테스트 검증값)
이유: 1봉 기울기는 단순 가격 노이즈에 반응. 5봉 기울기는 VPF 추세의 실제 방향 변화를 측정. 백테스트의 S1 승률 90.9%는 5봉 기울기 기준으로 검증된 수치.
[2] VPF 이중 스무딩 제거
// 현재: SMA(10) 후 EMA(sig_len) 추가 적용 → 이중 스무딩으로 값 지연 v_val = ta.sma(dist_n, 10) v_sig = ta.ema(v_val, sig_len) [v_val, v_sig] // vpf에 v_val 할당
// 수정: SMA(10) 단일 스무딩 (원본 VPF와 동일) v_val = ta.sma(dist_n, sig_len) v_val // vpf에 직접 반환
이유: 원본 VPF는 SMA 한 번만 적용. 이중 스무딩 시 Zone 진입/이탈 시점이 실제보다 지연되어 신호가 늦게 발생.
🟡 HIGH — 추후 구현 권장
진입 잠금 TP/SL (미구현): 개발 요구사항에 '필수'로 명시된 항목. 진입가·목표가·손절가 수기 입력 후 차트에 수평선 표시 + 패널에 현재 손익% 실시간 계산. 파라미터 그룹 G5 추가 필요.
🟢 유지 권장 (요구사항 외 추가 기능)
패널 위치 선택 / 박스 색상 커스터마이징 / VPF 배경색 3분할 → 모두 UX 개선에 유효. 단, VPF 배경 임계값(vpfThreshold) 기본값을 0.5 → 1.0으로 변경 권장 (Zone A 기준인 -1.0과 일치).

07
자주 헷갈리는 것들
Q. DN-BOX인데 배경이 빨개요. 매수해도 되나요?
원칙상 DN-BOX에서 신규 매수는 안 합니다. 단, S1 신호가 뜨는 순간은 UP-BOX로 전환되는 시점이라 DN-BOX와 동시에 뜰 수 없습니다. 빨간 배경 = Zone A 상태를 나타내는 것이고, 신호는 UP-BOX 전환 시에만 발생합니다.
Q. S1 신호가 뜨는데 FINAL SCORE가 50이에요. 들어가도 되나요?
S1 신호 + FINAL SCORE 50은 "위치는 괜찮지만 완벽하지 않음"입니다. EQS가 50 캡에 걸려 있는 경우(EMA20 아래)가 대부분입니다. 이럴 때는 절반 비중으로만 진입하고, RR⚠️RISK가 없는지 반드시 확인하세요.
Q. VPF SCORE가 N/A로 뜹니다.
데이터가 400봉(VPF Length 기본값) 미만이면 계산 불가입니다. 일봉 기준 약 1.6년치 데이터가 필요합니다. 차트를 더 오래된 기간으로 확장하거나, VPF Length를 낮추세요. N/A는 "0점"이 아니라 "측정 불가"입니다.
Q. TP/SL이 매 봉마다 바뀌는데 언제 기준으로 잡나요?
진입 시점의 TP/SL을 기록해서 그 값을 손절 기준으로 씁니다. 설정창 "진입 잠금"에 입력하면 차트에 수평선으로 고정됩니다. 이후 CANDIDATE가 유리한 방향으로 갱신되면 SL을 올려가는 트레일링 방식도 가능합니다.
Q. 연간 신호가 3~4건밖에 안 나와요. 너무 적지 않나요?
설계 의도에 맞는 빈도입니다. 거래 횟수를 늘리려면 Swing%를 낮추는 게 아니라 타임프레임을 낮추는 것이 정답입니다 (4H 사용 시 연 7건, 1H는 연 12건 수준). 또는 SOXL·TECL을 병행해 포트폴리오 전체 신호를 늘립니다. 하이브리드 전략은 신호 없는 달에도 월 적립이 진행되므로 "신호 대기 중 수익 기회 없음" 문제가 해소됩니다.
